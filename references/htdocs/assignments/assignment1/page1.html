Introduction to the Document Object Model

by kazuma sato

This page is a brief intoduction to the Document Object Model.

The Document Object Model (or DOM) is a convention for interacting and discribing objects in HTML, XHTML and XML documents. The DOM consists of a family tree stucture where parts of the documents, known as nodes, are organized.

Everything in an HTML document can be considered a node. Anywhere from the elements in the document, the attributes in the element to even whitespace between the elements as well. The following table is a list of the node types.

Node Type Name Discription
1 ELEMENT_NODE represents an element
2 ATTRIBUTE_NODE represents an attribute
3 TEXT_NODE represents textual content in an element or attribute
4 CDATA_SECTION_NODE represents a CDATA section in a document (text that will NOT be parsed by a parser)
5 ENTITY_REFERENCE_NODE represents an entity reference
6 ENTITY_NODE represents an entity
7 PROCESSING_INSTRUCTION_NODE represents a processing instruction
8 COMMENT_NODE represents a comment
9 DOCUMENT_NODE represents the entire document (the root-node of the DOM tree)
10 DOCUMENT_TYPE_NODE provides an interface to the entities defined for the document
11 DOCUMENT_FRAGMENT_NODE represents a "lightweight" Document object, which can hold a portion of a document
12 NOTATION_NODE represents a notation declared in the document type definition (DTD)

reference: Wikipedia - Document Object model

These are some terms used to discribe nodes as well.

  • Child: A child node is a node extending from another node. For example, a computer with internet access could be considered a child node of a node representing the internet. The inverse relationship is that of a parent node. If node C is a child of node A, then A is the parent node of C.
  • Degree: the degree of a node is the number of children of the node.
  • Depth: the depth of node A is the length of the path from A to the root node. The root node is said to have depth 0.
  • Edge: the connection between nodes.
  • Forest: a set of trees.
  • Height: the height of node A is the length of the longest path through children to a leaf node.
  • Internal node: a node with at least one child.
  • Leaf node: a node with no children.
  • Root node: a node distinguished from the rest of the tree nodes. Usually, it is depicted as the highest node of the tree.
  • Sibling nodes: these are nodes connected to the same parent node.

reference: Wikipedia - Node (computer science)

Tutorial

  1. Using document methods to access elements:
    console.log(document.getElementByTagName("p"));
    This will return a list of the all <p> tag elements in the document with all the node attributes. You can even access each element using document.getElementByTagName("p")[1] as an array.(note: node lists may look like an array but isn't really one. Not all methods which come with an array object can be used with a node list.)
